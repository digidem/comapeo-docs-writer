# CoMapeo Glossary

Definitions of terms used throughout the CoMapeo documentation.

## Terms

### About CoMapeo
A **CoMapeo Settings** option that opens the screen displaying information about CoMapeo, the current version installed, link to release updates, and a way to provide feedback.

### Activity
A coherent action with beginning and end (e.g., workshop, training trip). Definition depends on funder/project specifics.

### Agile Methodology
Project management framework that breaks projects down into smaller phases known as sprints, relying on iterative cycles.

### All Projects
A **Menu** option that displays a list of projects currently on the device. Any person using CoMapeo can create a **New Project** from this screen.

### APK (Android Package Kit)
File format used by Android phones to distribute and install mobile applications.

### App Shell
A skeleton version of the mobile app with only components necessary to launch, used for user testing.

### Audio (Audio Recording)
A feature that enables a person to record audio and attach it to an **Observation**.

### Bug
Errors or defects in code or design causing unexpected behavior.

### Camera
The third tab on the lower nav bar that opens the device’s camera to start a new **Observation**.

### Categories / Category
Pre-defined labels for organizing **Observations** within a **Project**.

### Categories Set
(Formerly Config) Files that contain and define the distinct **Categories** for a **Project**.

### Choose File
(Internally: File Picker) Feature enabling users to upload a custom Background Map to their device.

### Collaborators
Secondary term describing devices working on the same **Project**.

### CoMapeo Settings
(Formerly App Settings) **Menu** option for individual device preferences (units, background maps, **Early Access**). These apply across all **Projects** on the device.

### Connected
State when CoMapeo detects nearby devices on the same **Project** and router (or over internet).

### Coordinator
**Role** of a device with permission to adjust **Project** settings and invite devices.

### Coordinator Tools
(Formerly Project Settings) Screen where **Coordinators** adjust **Project Info**, **Categories Set**, **Remote Archive**, and Stats.

### Current Location
Map UI element indicating the device’s location.

### Current Project Card
**Menu** UI element indicating the active **Project**.

### Data
Information in digital form that can be transmitted or processed.

### Data & Privacy
Umbrella term for policies around gathering device and usage metric measurements.

### Data Continuity
Ability to maintain availability, integrity, and usability of data across different systems.

### Debug
Addressing things that are not working in software.

### Deliverable
A distinct item or outcome produced (e.g., publication, event), often for a partner or funder.

### Details
Customizable fields attached to a **Category** for gathering specifics about an **Observation**.

### Device
The supported hardware on which CoMapeo is installed.

### Device Data (Metrics)
Measurements about a device gathered with consent.

### Diagnostics
Anonymized info about device/app performance (crashes, errors) shared with consent. Never includes user data or precise location.

### Early Access
Term for pre-**Named Release** stages (MVP). Future: Opt-in feature for testing new capabilities.

### Event
A planned activity on specific dates with intended guests (e.g., workshop, meeting).

### Exchange
Feature enabling devices to mutually share **Observations** when connected (local or internet).

### Export
Feature enabling users to download and move **Observation** data outside the app.

### Feature Flag
Software concept enabling/disabling features without code modification (formerly "Experiments").

### Fixes (Bugfixes)
Changes made to software to correct a fault.

### Go to Current Location
Map UI element that centers the map on the device’s location.

### GPS Accuracy
Map UI element indicating GPS accuracy.

### Hotfix Release
Release addressing an urgent issue that cannot wait for a scheduled release.

### Inquiry
Individual or institution reaching out for conversation, training, or accompaniment.

### Internal Build
Build for testing specific features/fixes, not for release.

### Issues
Problems or challenges affecting product quality (bugs, feature requests).

### Map
Second tab on the lower nav bar displaying the area view.

### Mapeo 6
(Internal) Name for the new version of Mapeo Desktop/Mobile. Externally: CoMapeo.

### Mapeo Core
The peer-to-peer database that powers Mapeo.

### Mapping On Your Own
(Formerly Solo) Default experience of a device not yet collaborating on a **Project**.

### Menu
UI element (upper-right) opening the Menu UI.

### MVP (Minimum Viable Product)
Version with just enough features to be usable and provide feedback.

### Named Release
Major versions of CoMapeo with stable/integrated features (e.g., Abare).

### New Observation
Screen allowing a person to create a new **Observation**.

### Observation
Core feature: a digital note (point) at a specific location/time, potentially with photo/audio/details.

### Observation Metadata (GPS Info)
Screen displaying metadata for recorded GPS coordinates.

### Observations List
First tab on the lower nav bar displaying all available **Observations** for the **Project**.

### Participant
**Role** of a device with permission to contribute and **Exchange**.

### Partner
Entity with whom we have a reciprocal institutional relationship.

### Peer-to-peer (P2P)
Network connecting devices directly to share data.

### Photo Info
Screen displaying expanded **Observation** Photo and metadata.

### PR (Pull Request)
Mechanism for notifying readiness to merge new code.

### Process
Set of steps facilitating change.

### Project
Core feature enabling management of a collaborative territorial database.

### Project Info
Screen where **Coordinators** edit project name, description, and color.

### Project Invite
Encrypted invites using a secure project key to add devices.

### QA (Quality Assurance)
Process of testing and verifying product quality.

### Release
Latest available grouping of app updates.

### Remote Archive
(Formerly Mapeo Cloud) Dedicated private server for sharing data over internet.

### Role
Description of user types (e.g., **Coordinator**, **Participant**).

### Scale
Map UI element displaying current map scale.

### Share
Feature enabling sending **Observation** data/metadata as text outside the app.

### Sprint
Short, time-boxed period to complete a set amount of work.

### Team
Group of devices connected and participating on the same **Project**.

### Tracks
Fourth tab on the lower nav bar for recording points along a path (georeferenced track).

### User Requirement
Specific technical details on how a product should work.

### User Story
Requirement phrased from user perspective ("As a X, I want Y, so that Z").

### Validated
Descriptor for GPS/photos used to strengthen proof of evidence.

### YOLO Merge
Merging code without review (risky).
